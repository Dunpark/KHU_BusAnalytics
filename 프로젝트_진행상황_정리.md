# 젠트리피케이션 & 폐업 위험도 조기 경보 시스템 프로젝트 진행 상황 정리

**작성일:** 2025년  
**프로젝트 상태:** 절차 4 완료, 절차 5 대기 중  
**최종 업데이트:** 절차 4 데이터셋 정리 완료

## 📋 프로젝트 개요

### 프로젝트 주제
**젠트리피케이션 & 폐업 위험도 조기 경보 시스템**

### 분석 단위
- **기본 단위:** 서울시 내 매장별
- **집계/분석 단위:** 상권(또는 상권배후지) × 업종 × 분기

### 분석 기간
**2019Q1 ~ 2024Q4** (6년, 24분기)
- 코로나19 전·후 비교 가능
- 계절성, 트렌드 변화, 외부 충격 영향 분석 가능

### 프로젝트 목표

#### 목표 1: 폐업 요인 탐색 및 패턴 분석
- 폐업한 매장들을 분석하여 폐업 요인들을 탐색
- 폐업 전 징후(패턴) 확인
- **활용 지표:**
  - 상권 특성
  - 유동/생활 인구
  - 임대료/공실률 (젠트리피케이션 관찰)
  - 배후지 인구/소득/소비 구조
  - SNS/트렌드 Hot Place 지표
- **세분화 분석:**
  - 업종유형별 파티션
  - 지역별 파티션
  - 코로나 전·후 비교

#### 목표 2: 운영 중 매장의 폐업 위험도 점수화
- 현재 서울시에서 운영 중인 매장들을 대상으로 폐업 위험도 점수화
- **최종 산출물:** 매장별 폐업 위험도 점수 및 관련 통계를 **대시보드 형태로 시각화**하여 제공

#### 목표 3: 실무 가이드라인 제시
- 매장별 창업/대출 리스크 가이드라인 제시

## 📚 프로젝트 요구사항 및 평가 기준

### Final Project 요구사항 (절차 1에서 분석 완료)

#### 보고서/발표에서 답해야 하는 질문
1. **Why do I care?** - 명확한 동기와 실무적 결과 필요
2. **Are the answers obvious?** - 자명하지 않은 분석 결과 필요
3. **Are the arguments believable?** - 통계, 시각화, 실제 데이터, 다양한 분석 기법으로 뒷받침

#### 프로젝트 구조 (6단계)

1. **Business Understanding**
   - 해결할 비즈니스 문제 식별

2. **Data Understanding**
   - 수집/생성한 데이터와 출처 설명

3. **Data Preparation**
   - 데이터 마이닝에 필요한 형식으로 통합하는 방법 명시

4. **Modeling**
   - 데이터 마이닝 모델 선택(단일 또는 다중)
   - 대안과 장단점 논의
   - 비즈니스 문제 해결 방식과 이유 설명

5. **Evaluation**
   - 데이터 마이닝 결과 평가 방법
   - 비즈니스 케이스 제시(예상 개선 효과, ROI 등)
   - 대안이 있다면 제시

6. **Deployment**
   - 결과 배포 방법 및 비즈니스 문제 해결 방식
   - 배포 시 주의사항
   - 관련 리스크와 완화 방안
   - 관련 당사자(기업/개인)에 대한 시사점

### 주요 평가 기준
- 명확한 동기와 실무적 결과
- 통계, 시각화, 실제 데이터 기반 분석
- 다양한 분석 기법 활용
- 설득력 있는 논증과 결과 제시

## 🛠️ 활용 가능한 도구 및 기법 (절차 2에서 분석 완료)

### 도구
- R, Python, Tableau, Orange3

### 활용 가능한 개념 및 기법

1. **탐색적 데이터 분석 (EDA)**
   - 상관성 분석, 히트맵 등
   - 데이터 패턴과 관계 파악

2. **시각화**
   - Tableau 활용
   - 인사이트 전달에 중요

3. **데이터 마이닝**
   - 분류, 클러스터링, 회귀 등

4. **평가 (Evaluation)**
   - Confusion Matrix
   - Overfitting 평가
   - 모델 성능 측정

5. **데이터 전처리**
   - Scaling
   - 결측치 처리
   - 이상치 확인 및 제거

6. **분류 기법**
   - Classification Tree
   - KNN (K-Nearest Neighbors)

7. **클러스터링**
   - 그룹화 및 패턴 발견

8. **회귀 분석**
   - Explanatory Regression
   - Regression
   - Regression_Logit

9. **실험 설계**
   - AB Test

### 기법 선택 원칙
- **인사이트 도출이 최우선**
- 모든 기법을 다 활용할 필요 없음
- 분석 환경과 상황에 적절한 기법 선택

## 📊 프로젝트 방향성 이해 (절차 3에서 분석 완료)

### 데이터 마이닝 접근
- **분류:** 폐업/영업 중 이진 분류 또는 위험도 등급 분류
- **회귀:** 위험도 점수 예측
- **클러스터링:** 유사 패턴 그룹화
- **EDA:** 폐업 요인 탐색

### 젠트리피케이션 분석
- 임대료/공실률 추이 분석
- 상권 변화 패턴 식별
- 폐업과의 인과관계 탐색

### 다차원 분석
- 업종별, 지역별, 시기별 차이 분석
- 코로나 전·후 비교
- 상호작용 효과 분석

### 시각화 및 대시보드
- Tableau 활용
- 매장별 폐업 위험도 점수 시각화
- 관련 통계치를 대시보드로 제공
- 인사이트 전달에 중점

### 예상 분석 흐름
1. 폐업 매장 데이터 수집 및 전처리
2. EDA: 폐업 요인 탐색, 상관성 분석
3. 모델링: 분류/회귀 모델로 위험도 예측
4. 평가: 모델 성능 검증, 비즈니스 케이스 제시
5. 배포: 위험도 점수 대시보드 시스템, 가이드라인 제시

### 주요 도전 과제 및 고려사항
1. 데이터 수집: 매장별 상세 데이터, 상권 데이터, 인구/소득 데이터 통합
2. 젠트리피케이션 측정: 임대료/공실률 데이터의 시계열 분석
3. 외생 변수: 코로나 등 외부 충격의 영향 분리
4. 해석 가능성: 실무 적용을 위한 모델 해석성 확보
5. 대시보드 설계: 매장별 위험도 점수와 관련 통계를 직관적으로 표현
